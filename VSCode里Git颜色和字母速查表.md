# VS Code 里 Git 颜色和字母速查表

## 1. 这份表是干什么的

当你在 VS Code 资源管理器里看到文件旁边出现：

- `U`
- `M`
- `A`
- 绿色
- 黄色
- 红色

最容易困惑的是：

`这些到底是在说文件怎么了？`

这份速查表就是帮你把：

- VS Code 里的视觉提示
- Git 里的真实状态

对应起来。

---

## 2. 最先记住的一句话

如果你对颜色不确定，永远以这个命令为准：

```powershell
git status
```

因为：

- VS Code 的颜色会受主题影响
- 但 `git status` 的语义最准确

---

## 3. 常见字母是什么意思

### `U`

含义：

`Untracked`

意思是：

- 文件存在
- 但 Git 还没有开始跟踪它

常见场景：

- 你新建了一个文件
- 但还没有执行 `git add`

---

### `M`

含义：

`Modified`

意思是：

- 这个文件以前已经被 Git 跟踪过
- 你后来又改了它

常见场景：

- 改了一行代码
- 改了一段文档
- 改了配置内容

---

### `A`

含义：

`Added`

意思是：

- 这个文件已经被加入暂存区
- 准备在下一次 commit 里提交

常见场景：

- 新文件先 `git add`
- 或原来没跟踪的文件第一次被纳入 Git

---

### `D`

含义：

`Deleted`

意思是：

- 文件被删掉了
- Git 检测到了删除动作

---

### `R`

含义：

`Renamed`

意思是：

- 文件被重命名了

---

## 4. 常见颜色怎么理解

注意：

不同 VS Code 主题颜色会有差异，但大致规律差不多。

### 绿色

通常表示：

- 新文件
- 新增内容
- 还没提交的新变化

经常会和这些状态对应：

- `U`
- `A`

你可以先简单记成：

`绿色 = 新增类变化`

---

### 黄色 / 橙色

通常表示：

- 已经被跟踪的文件被修改了

经常会和这些状态对应：

- `M`

你可以先简单记成：

`黄色 = 改过的旧文件`

---

### 红色

通常表示：

- 删除
- 冲突
- 或某种需要注意的问题状态

最常见是：

- `D`

---

### 白色 / 普通颜色

通常表示：

- 当前没有未提交改动
- 文件是“干净”的

也就是：

`Git 觉得这个文件现在和最近一次提交一致`

---

## 5. 和 Git 状态怎么对应

### 场景 1：你新建了文件，但还没 `git add`

你可能看到：

- `U`
- 绿色

Git 真实意思：

`untracked`

---

### 场景 2：你改了旧文件，但还没 `git add`

你可能看到：

- `M`
- 黄色

Git 真实意思：

`modified but not staged`

---

### 场景 3：你已经 `git add`，但还没 `git commit`

你可能看到：

- `A`
- 或不同主题下的已暂存样式

Git 真实意思：

`changes to be committed`

---

### 场景 4：你已经 `git commit`

你通常会看到：

- 文件恢复成普通白色
- 没有状态字母

Git 真实意思：

`working tree clean（至少这个文件是干净的）`

---

## 6. 最实用的判断方法

如果你在 VS Code 里看到颜色或字母，不确定什么意思，按这个顺序判断：

1. 先看文件是不是新文件
2. 再想它有没有被改过
3. 再执行：

```powershell
git status
```

4. 如果还想看具体改动，再执行：

```powershell
git diff
git diff --staged
```

---

## 7. 你现在最常见的 4 种状态

结合你 Day21 和 Day22 的练习，你现在最常见的是这 4 种：

### `U`

新文件，还没 add

### `M`

旧文件改了，还没 add

### `A`

已经 add，准备 commit

### 白色

已经 commit，当前干净

只要先把这 4 种看熟，你就已经能看懂大多数 VS Code 里的 Git 提示了。

---

## 8. 一句话总结

VS Code 里的颜色和字母，本质上都是在帮你快速回答一个问题：

`这个文件现在是新建的、修改过的、已暂存的，还是已经干净了？`
