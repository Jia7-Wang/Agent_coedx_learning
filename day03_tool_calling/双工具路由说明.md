# 双工具路由说明

## 1. 这个文件是干什么的

它配合 `multi_tool_routing.py` 使用，帮助你理解：

- 当模型面前有多个工具时，会如何选择
- 模型是只选一个，还是可以连续选多个
- 程序如何逐个执行工具并把结果回传

---

## 2. 这次有哪两个工具

### 工具 1：`get_learning_plan_by_level`

作用：

- 判断用户适合哪一阶段的学习路线

### 工具 2：`get_engineering_basics_topics`

作用：

- 返回适合穿插学习的公司工程基础主题

---

## 3. 这一步和单工具调用的区别

单工具版本重点是：

- 理解工具调用闭环

双工具版本重点是：

- 理解模型如何在多个工具之间做路由选择

也就是说，模型不仅要判断“要不要调用工具”，还要判断：

- 调哪个工具
- 调几个工具
- 调用顺序是什么

---

## 4. 你运行时重点看什么

运行：

```powershell
python .\multi_tool_routing.py
```

重点关注输出里的这几项：

- `Tool calls: ...`
- `Selected tools:`
- 每个工具的 `arguments`
- 每个工具的 `result`
- `Final answer after tool routing:`

---

## 5. 你现在最值得观察的现象

### 情况 1：模型只调用一个工具

说明模型认为一个工具就足够回答问题。

### 情况 2：模型调用两个工具

说明模型认为：

- 既要给学习路线
- 也要补工程常识

这是更接近真实 Agent 工作方式的情况。

---

## 6. 一句话总结

`单工具调用在学闭环，双工具调用在学路由。`
