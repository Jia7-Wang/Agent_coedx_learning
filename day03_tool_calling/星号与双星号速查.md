# `*` 与 `**` 速查

## 1. 最短记忆版

- `*`：展开列表 / 元组
- `**`：展开字典

你可以简单记成：

- `*` 拆一串值
- `**` 拆一组键值对

---

## 2. `*` 是什么意思

### 2.1 用在列表里

```python
nums = [1, 2, 3]

result = [0, *nums, 4]
```

等价于：

```python
result = [0, 1, 2, 3, 4]
```

意思是：

`把 nums 里的元素一个个展开，放进新列表里`

### 2.2 用在函数调用里

```python
nums = [1, 2, 3]
print(*nums)
```

等价于：

```python
print(1, 2, 3)
```

---

## 3. `**` 是什么意思

### 3.1 用在函数调用里

```python
data = {"name": "Tom", "age": 18}

def show(name, age):
    print(name, age)

show(**data)
```

等价于：

```python
show(name="Tom", age=18)
```

意思是：

`把字典里的键值对拆开，当成命名参数传进去`

### 3.2 用在字典里

```python
a = {"x": 1}
b = {"y": 2}

c = {**a, **b}
```

结果是：

```python
{"x": 1, "y": 2}
```

意思是：

`把两个字典展开后合并`

## 4. 你当前代码里的真实例子

### 4.1 `*tool_messages`

在 [multi_tool_routing.py](</d:/PythonProjects/Agent_coedx_learning/day03_tool_calling/multi_tool_routing.py>) 里：

```python
messages = [
    {...},
    {...},
    *tool_messages,
]
```

意思是：

`把 tool_messages 列表里的每一个工具消息，逐个展开到 messages 里`

如果不写 `*`，就会变成“列表里嵌套一个列表”，通常不是我们想要的。

### 4.2 `OpenAI(**client_kwargs)`

在 Day 3 脚本里：

```python
client_kwargs = {
    "api_key": api_key,
    "base_url": base_url,
}

client = OpenAI(**client_kwargs)
```

等价于：

```python
client = OpenAI(
    api_key=api_key,
    base_url=base_url,
)
```

意思是：

`把字典里的键值对拆成函数参数`

---

## 5. 最容易混淆的点

### `*`

通常处理：

- 列表
- 元组

### `**`

通常处理：

- 字典

所以你可以直接背：

- `*` 解包列表
- `**` 解包字典

---

## 6. 一句话总结

`*` 是把一串值展开，`**` 是把一组键值对展开。
