# 工程版说明

## 1. 这份脚本是干什么的

`engineering_tool_calling.py` 是在 Day 3 教学版基础上做的“更接近真实项目”的版本。

它不是为了让你先学更多新概念，而是让你看到：

- 参数校验为什么重要
- 工具失败为什么不能直接让程序崩掉
- 为什么真实项目需要日志

---

## 2. 相比教学版多了什么

### 1. 参数校验

脚本会检查：

- `level` 是否存在
- `level` 是否是字符串
- `level` 是否在允许值范围内

### 2. 参数解析保护

如果模型传回来的工具参数不是合法 JSON，脚本会报出清晰错误，而不是直接崩在 `json.loads()`。

### 3. 工具执行异常处理

即使某个工具执行失败，脚本也会：

- 记录错误
- 把失败信息作为工具结果回传模型
- 尽量继续完成最终回答

### 4. 结构化日志

脚本会打印 JSON 格式的日志事件，例如：

- `request.start`
- `response.first`
- `tool.requested`
- `tool.success`
- `tool.error`
- `response.final`

这就是最基础的可观测性思维。

---

## 3. 你运行时重点看什么

运行：

```powershell
python .\engineering_tool_calling.py
```

重点看：

- 每个 `log_event(...)` 打印出来的 JSON
- 工具参数是否先被校验
- 工具失败时程序是否还能继续

---
