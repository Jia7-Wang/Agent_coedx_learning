# 90天详细到天的 Agent 学习计划

## 1. 这版计划怎么用

这版是重新整理后的 90 天计划，重点解决两个问题：

1. 前 20 天严格对齐你已经完成的 `Day1 - Day20`
2. 从 Day21 开始，补上现在公司项目里真实会用到的技术栈：
   - Git
   - LangChain
   - LangGraph
   - MCP / Skills / Tool 生态
   - FastAPI
   - Pydantic
   - PostgreSQL / pgvector
   - Redis
   - pytest
   - tracing / logging
   - Docker / Docker Compose

建议学习节奏：

- 每天 1.5 到 3 小时
- 每周学习 6 天，1 天轻复盘
- 每天都要有“最小产出”

---

## 2. 90 天后的目标

如果你认真执行完这 90 天，你应该能够：

- 独立写基础 Agent Demo
- 理解并解释 Tool Calling、RAG、State、Workflow、Session
- 做出带 retry / dead letter / human handoff / human resolution 的工作流 Agent
- 用 LangChain / LangGraph 做更贴近真实项目的 Agent
- 用 FastAPI 把 Agent 包装成服务
- 用 Pydantic 管理输入输出和配置
- 用 PostgreSQL / pgvector 做基础 RAG 存储
- 用 Git 做基本协作
- 用 Docker 跑本地服务
- 做出 2 到 3 个能展示的项目

---

## 8. Git 这条线你至少要学到什么

- `git diff`
- `git log --oneline`
- `.gitignore`

- 合并分支

### 建议会

- 看懂 Pull Request

- 知道为什么提交前要先看 diff

- rebase
- squash
- cherry-pick
- stash
- revert

## 9. 每周固定动作

每周建议固定做 4 件事：

1. 复盘 1 次：写本周学会了什么。
2. 提交 1 次：至少保留 1 次有效 commit。
3. 讲解 1 次：尝试向“未来的自己”讲清一个概念。
4. 整理 1 次：把命令、报错、知识点整理进笔记。

## 10. 这版计划的核心逻辑

这版不是“先学一堆框架名词”，而是：

1. 先用 Day1-Day20 把 Agent 基础工作流吃透
2. 再补 Git、FastAPI、Pydantic、Docker、测试这些工程基础
3. 再上 LangChain、LangGraph、MCP，把前面的概念映射到真实技术栈
4. 最后用作品集把知识真正固化

这样你学出来，不只是“知道一些词”，而是真的更接近公司里能落地的人。
