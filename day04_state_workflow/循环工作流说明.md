# 循环工作流说明

## 1. 这份脚本是干什么的

`workflow_loop_planner.py` 是 Day 4 的升级版。

它和前一个版本最大的区别是：

- 不是只跑两段
- 而是让 state 连续经过 3 个步骤

这样你会更明显地看到 workflow 的推进过程。

---

## 2. 它一共跑哪 3 步

### 第 1 步：`break_down_goal`

把用户目标拆成多个子任务。

### 第 2 步：`suggest_execution_order`

根据子任务，给出执行顺序。

### 第 3 步：`plan_week1`

根据执行顺序，生成第一周计划。

---

## 3. 这一步比前一个版本多了什么

前一个版本重点是：

- 理解 state 会被更新

这个版本重点是：

- 理解 state 会跨多个步骤持续流动
- 每一步都依赖上一步写入的结果

也就是说，真正的 workflow 感更明显了。

---

## 4. 你运行时重点看什么

运行：

```powershell
python .\workflow_loop_planner.py
```

重点看：

- `State after step 1`
- `State after step 2`
- `Final state`

你会看到同一个 `state` 对象在不断长出新字段内容。

---

## 5. 一句话总结

`两步版在学 state 会更新，循环版在学 state 会持续推进 workflow。`
