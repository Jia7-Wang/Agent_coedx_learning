# 为什么 Day 7 会多次检索

## 1. 最短答案

Day 7 会多次检索，不是因为代码里某一行“强制它查两次”，而是因为：

- 模型面对更复杂的任务目标
- `tool_choice="auto"` 允许模型自己决定调用次数
- 程序支持处理多个 `tool_calls`

所以最后是：

`模型自己决定查两次，代码负责把这两次都执行出来。`

---

## 2. 决定“查几次”的是谁

在 [integrated_learning_agent.py](</d:/PythonProjects/Agent_coedx_learning/day7_integrated_agent/integrated_learning_agent.py>) 里，第一轮请求有这句：

```python
tool_choice="auto"
```

它的意思是：

`让模型自己决定要不要调用工具，以及调用几次。`

所以：

- 可能 0 次
- 可能 1 次
- 可能 2 次

并不是程序硬编码成“一定查两次”。

---

## 3. 为什么 Day 7 比 Day 6 更容易查两次

### Day 6 的目标

Day 6 更像：

- 回答一个知识问题

例如：

- 应该优先补哪些工程基础

这类问题通常查一次知识库就够了。

### Day 7 的目标

Day 7 更像：

- 基于知识库给出一个行动方案

而且 prompt 里明确要求：

- 学习重点
- 建议顺序
- 一个最小项目建议

这比单纯问答更复杂，所以模型更可能从不同角度多查几次。

---

## 4. 代码里真正“支持多次检索”的地方

虽然不是代码命令它查两次，但程序必须有能力接住多个工具调用。

关键就是这段：

```python
for tool_call in message.tool_calls:
    ...
```

它的作用是：

- 如果模型返回 1 个工具调用，就执行 1 次
- 如果模型返回 2 个工具调用，就执行 2 次
- 如果模型返回更多，就继续逐个执行

如果你只写：

```python
tool_call = message.tool_calls[0]
```

那即使模型想查两次，程序也只会执行第一次。

---

## 5. 任务复杂度为什么会影响检索次数

模型在 Day 7 里，面对的不是一个单点问题，而是一个组合目标：

- 我要学 Agent
- 我要补 Git / 调试 / Docker
- 我要在 6 周内做出一个简单项目
- 你还要给我下一步行动方案

这类任务通常会让模型隐式拆成几部分：

- 学习路线
- 工程基础
- 项目实践
- 行动安排

所以它更有可能用多个 query 去查。

---

## 6. 一句话总结

Day 7 会多次检索的根本原因是：

`任务更复杂，模型在 auto 模式下自己决定发起多个 tool_calls；而程序又支持逐个执行这些 tool_calls。`
