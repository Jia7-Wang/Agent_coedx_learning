## 2. 什么是结构化输出

普通调用时，模型可能返回一段自然语言：

```text
这个用户是一位想学习 Agent 的新手，建议先学习 Python、API、工具调用。
```

这段话人能看懂，但程序不容易稳定处理。

结构化输出的目标是让模型返回类似这样的内容：

```json
{
  "level": "beginner",
  "goal": "learn_agent",
  "next_step": "study_tool_calling",
  "topics": ["python", "api", "prompt", "tools"]
}
```

这样做的好处是：

- 程序更容易解析
- 结果更容易存数据库
- 更适合做后续流程判断
- 更适合接工具和工作流

---

## 3. 今天这个练习会做什么

我们会让模型读取一段“用户需求”，然后输出固定 JSON。

例如输入是：

```text
我是零基础，想学 Agent 开发，平时每天能学 2 小时，希望 3 个月内能做项目。
```

模型要输出：

- 学习者等级
- 学习目标
- 每日投入时间
- 建议起点
- 推荐主题列表

这就是最基础的“信息提取型结构化输出”。

## 6. 你今天重点要看懂什么

### 6.1 Prompt 不只是“提问”

今天你会看到，prompt 里不仅要写任务，还要写：

- 返回格式
- 字段要求
- 不能输出什么

### 6.2 解析结果是工程关键

真正做 Agent 时，你经常要做这件事：

1. 模型输出结构化结果
2. Python 读取结果
3. 程序根据字段做下一步判断

例如：

- 如果 `level=beginner`，走新手学习路线
- 如果 `need_tools=true`，进入工具调用流程
- 如果 `risk=high`，触发人工确认

### 6.3 结构化输出是工具调用前的重要基础

后面你学 tool calling 时会发现，它本质上也是一种“结构化结果”。

所以今天这一步非常关键。

## 8. 常见报错怎么理解

### 8.1 返回内容不是合法 JSON

说明：

- 模型没有完全遵守格式
- 或中转平台对格式支持不稳定

解决：

- 先加强 prompt
- 再做最小清洗
- 必要时加重试

### 8.2 模型名不支持

说明：

- 中转平台不支持你当前填的模型名

解决：

- 改成中转平台真实支持的模型名

### 8.3 Base URL 写错

说明：

- 请求没有正确发到中转地址

解决：

- 检查是否带 `/v1`
- 检查域名是否正确
