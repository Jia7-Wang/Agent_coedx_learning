Agent 学习路线建议：
零基础学习 Agent 时，建议先打牢 Python、API、JSON 和结构化输出基础，再学习 Tool Calling、状态管理、工作流和 RAG。不要一开始就追求多 Agent 编排。

工程基础建议：
如果目标是以后进入公司项目，学习 Agent 的同时要补 Git、日志与调试、项目目录结构、API 联调和 Docker。这些内容能帮助你从“会写 demo”走向“能参与真实项目”。

状态历史建议：
当 Agent 不只需要读取当前 state，还需要保留每一次更新痕迹时，可以维护 history 或 session trace。这样能知道状态是如何一步步推进的，也更方便调试、复盘和多轮协作。

持久化建议：
如果希望 Agent 在多次运行之间保留进度，就不能只把 state 放在内存里，而要把它保存到文件、数据库或其他外部存储。这样下次启动时才能恢复之前的 session。

多会话建议：
当同一个 Agent 需要同时服务多个用户、多个学习目标，或者同一用户的多个任务时，就需要多 session 管理。常见做法是用 session_id 区分不同状态，再按 session_id 分开存储和读取。
