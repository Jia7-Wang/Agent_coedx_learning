## 4. 先把几个核心概念讲清楚

### 什么是 API

API 你可以先理解成：

`别人和你的程序说话的一种入口。`

比如：

- 浏览器访问一个地址
- 前端发来一个请求
- 另一个 Python 服务调用你的接口

这些都属于“通过 API 使用你的程序能力”。

### 什么是 FastAPI

FastAPI 是一个用来写 Python Web API 的框架。

对你现在这个阶段来说，先记住它最重要的几个特点：

- 写法相对清楚
- 很适合做 JSON 接口
- 后面和 Pydantic、测试、文档都很容易结合

它非常适合你后面把 Agent 包成服务。

### 什么是路由

路由可以理解成：

`不同的 URL 路径，对应不同的处理函数。`

例如：

- `/` 处理首页或根接口
- `/health` 处理健康检查
- `/chat` 处理聊天请求

你以后做 Agent API 时，本质上也是在不断增加这种“路径 -> 处理逻辑”的映射。

### 什么是请求和响应

请求就是别人发给你服务的信息。

响应就是你的服务返回给别人的结果。

今天先用最简单的 GET 接口练习，不引入复杂请求体。

所以你只需要先接受这个最小模型：

`客户端请求一个路径，服务返回一段 JSON。`

### 为什么几乎每个服务都有 `/health`

`/health` 是健康检查接口。

它通常用来回答：

- 服务有没有启动成功
- 服务现在能不能响应
- 部署后容器是不是活着

在本地你会觉得它很简单，但到了公司里它很重要，因为：

- Docker / Compose 常常会看它
- 部署平台会看它
- 运维和监控系统也会看它

所以今天学 `/health`，不是为了接口本身复杂，而是为了建立“服务化”的思维。

## 7. 今天怎么练

先进入仓库根目录：

```powershell
cd d:\PythonProjects\Agent_coedx_learning
```

### 第一步：先看代码结构

打开：

- `day26_fastapi_basics_lab/main.py`

你会看到今天只有两个接口：

- `/`
- `/health`

### 第二步：启动服务

先推荐用 uvicorn 启动：

```powershell
uvicorn day26_fastapi_basics_lab.main:app --reload
```

如果你更想直接运行脚本，也可以：

```powershell
python .\day26_fastapi_basics_lab\main.py
```

### 第三步：在浏览器访问

启动成功后，打开浏览器访问：

- `http://127.0.0.1:8000/`
- `http://127.0.0.1:8000/health`

你应该能看到 JSON 返回。

### 第四步：观察自动文档

FastAPI 很适合初学者的一个点是，它自带接口文档。

启动服务后访问：

- `http://127.0.0.1:8000/docs`

你会看到 Swagger UI。

这也是为什么很多团队喜欢用 FastAPI 做内部服务原型。

## 8. 今天建议你重点观察什么

### 观察 1：脚本和服务的区别

脚本通常是：

- 你手动运行一次
- 执行完就结束

服务通常是：

- 启动后持续运行
- 等别人来请求
- 每次请求都返回响应

这是你从“写 demo”走向“写服务”的关键一步。

### 观察 2：接口返回的是结构化数据

今天返回的内容是 JSON 风格的数据。

这和你前面做 structured output 的感觉会慢慢接上：

- 不是只给人看
- 也是给程序看

### 观察 3：`/health` 虽然简单，但很工程化

它不是炫技接口，却是非常真实的工程接口。

很多公司项目里，第一个稳定存在的接口就是它。

## 9. 今天最容易卡住的点

### 1. 以为 FastAPI 就是“会写网站”

不完全是。

你现在学它，重点不是页面，而是：

`把 Python 能力封装成 HTTP 服务。`

### 2. 以为接口越复杂越说明学到了东西

不是。

今天故意只做最小接口，是为了让你先把：

- 路由
- 启动
- 请求地址
- JSON 响应

这几个最核心概念打牢。

### 3. 看到 `uvicorn module:app` 这种写法会懵

这是正常的。

你先这样理解：

- `day26_fastapi_basics_lab.main` 是 Python 模块路径
- `app` 是这个模块里的 FastAPI 应用对象

也就是：

`让 uvicorn 去启动 main.py 里的 app`
